📝 Ajustes no web.php

Integração do Módulo Vendas

📋 Resumo das Mudanças

  • Corrigido o registro do módulo vendas (de Vendas para Module)
  • Atualizado o component user para usar app\modules\vendas\models\Usuario
  • Adicionadas rotas específicas do módulo vendas no urlManager
  • Ajustado o AccessControl para permitir ações públicas do módulo
  • Mantidas todas as suas configurações e módulos existentes

🔧 Mudança 1: Registro do Módulo

Problema: A classe do módulo estava incorreta

❌ ANTES (Incorreto)

'vendas' => [
    'class' => 'app\modules\vendas\Vendas',
],

✅ DEPOIS (Correto)

'vendas' => [
    'class' => 'app\modules\vendas\Module',
],
ℹ️ Explicação: A classe do módulo deve se chamar Module (padrão do Yii2), não Vendas.

👤 Mudança 2: Component User

Problema: O identityClass estava apontando para o model antigo

❌ ANTES

'user' => [
    'identityClass' => 'app\models\Users',
    'enableAutoLogin' => true,
    'loginUrl' => ['login/index'],
    'authTimeout' => 1800,
],

✅ DEPOIS

'user' => [
    'identityClass' => 'app\modules\vendas\models\Usuario',
    'enableAutoLogin' => true,
    'loginUrl' => ['login/index'],
    'authTimeout' => 1800,
],
⚠️ IMPORTANTE: Mantive o loginUrl apontando para seu controller atual (login/index). Se quiser usar o login do módulo vendas, altere para ['vendas/site/login'].
✓ O que isso faz:
• Agora o Yii2 usará o model Usuario do módulo vendas para autenticação
• O model Usuario já implementa IdentityInterface corretamente
• Suporta hash de senha com bcrypt
• Tem métodos findByEmail() e validatePassword()

🔗 Mudança 3: Rotas do Módulo

Adicionei rotas específicas para o módulo vendas 
'rules' => [
    // Suas rotas originais (mantidas)
    'signup' => 'signup/index',
    'login' => 'login/index',
    'logout' => 'login/logout',
    'profile' => 'profile/index',
    
    // Rotas do módulo vendas (NOVAS)

    'vendas/login' => 'vendas/site/login',

],

🔒 Mudança 4: Access Control

Adicionei exceções para permitir acesso público ao login do módulo 
'as beforeRequest' => [
    'class' => 'yii\filters\AccessControl',
    'rules' => [
        [
            'actions' => ['login', 'error', 'forgot', 'register', 'resend', 'signup'],
            'allow' => true,
        ],
        [
            'allow' => true,
            'roles' => ['@'],
        ],
    ],
    // Exceções para ações do módulo vendas
    'except' => [
        'vendas/site/login',
        'vendas/site/error',
    ],
],
ℹ️ O que isso faz:
• Permite acesso público ao login do módulo vendas
• Permite acesso à página de erro do módulo
• Mantém suas regras de acesso originais

⚠️ Considerações Importantes

🔴 Migração do Model Users

Você precisa decidir:

  1. Manter os dois sistemas de autenticação (atual + vendas)?
  2. Migrar completamente para o model Usuario do módulo vendas?
  3. Criar compatibilidade entre os dois models?

Opção 1: Dois Sistemas Paralelos

Vantagem: Não quebra nada do sistema atual
Como usar:
  • Login atual: /login (usa app\models\Users)
  • Login vendas: /vendas/login (usa app\modules\vendas\models\Usuario)
⚠️ Problema: Usuários diferentes em cada sistema

Opção 2: Migração Completa (Recomendado)

Vantagem: Sistema unificado
Passos necessários:
  1. Migrar dados da tabela users para prest_usuarios
  2. Atualizar todos os controllers que usam app\models\Users
  3. Testar completamente o sistema

Opção 3: Adapter (Ponte entre os dois)

Vantagem: Compatibilidade temporária
Como fazer: Criar um adapter que faz Usuario ler da tabela users
Quando usar: Transição gradual
