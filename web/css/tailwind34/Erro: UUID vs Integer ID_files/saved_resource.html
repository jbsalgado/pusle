🔴 Erro: UUID vs Integer ID

Solução para incompatibilidade de tipos de ID

🔍 Entendendo o Erro

SQLSTATE[22P02]: Invalid text representation: 7 ERRO: sintaxe de entrada é inválida para tipo uuid: "1" SELECT * FROM "prest_usuarios" WHERE "id"=1

💡 O que está acontecendo?

  1. Causa: Há uma sessão ativa com ID inteiro (1) do sistema antigo
  2. Problema: A nova tabela prest_usuarios usa UUID como ID
  3. Resultado: PostgreSQL não consegue converter "1" em UUID
⚠️ Por que isso aconteceu:
Quando você alterou o identityClass no web.php de app\models\Users para app\modules\vendas\models\Usuario, o Yii2 tentou carregar o usuário da sessão antiga usando o ID que estava salvo (que era um inteiro).

⚡ Solução Rápida (Recomendada)

1Limpar Todas as Sessões

A forma mais rápida é fazer logout de todos os usuários e limpar as sessões.

Opção A: Limpar Sessões via Banco de Dados 
-- Se você armazena sessões no banco (tabela 'session')
TRUNCATE TABLE session;

-- Ou se usar outro nome de tabela
TRUNCATE TABLE sessions;
Opção B: Limpar Sessões via Arquivo 
// Deletar arquivos de sessão do diretório runtime
rm -rf runtime/session/*

// No Windows (PowerShell)
Remove-Item -Path runtime\session\* -Force
Opção C: Limpar Sessões Programaticamente

Crie um comando Yii2 temporário:

// app/commands/ClearSessionsCommand.php
namespace app\commands;

use Yii;
use yii\console\Controller;

class ClearSessionsController extends Controller
{
    public function actionIndex()
    {
        // Limpar diretório de sessões
        $sessionPath = Yii::getAlias('@runtime/session');
        if (is_dir($sessionPath)) {
            $files = glob($sessionPath . '/*');
            foreach ($files as $file) {
                if (is_file($file)) {
                    unlink($file);
                }
            }
            echo "Sessões limpas com sucesso!\n";
        }
        
        // Se usar sessões no banco
        if (Yii::$app->db->schema->getTableSchema('session')) {
            Yii::$app->db->createCommand()->truncateTable('session')->execute();
            echo "Tabela de sessões limpa!\n";
        }
    }
}

Execute:

php yii clear-sessions
⚠️ Atenção: Todas as sessões ativas serão perdidas. Todos os usuários precisarão fazer login novamente.

🛠️ Solução Completa (Ajustar o Model Usuario)

2Ajustar o Model Usuario para Lidar com Sessões Antigas

Adicione validação no model para ignorar IDs inválidos.

Edite o arquivo app/modules/vendas/models/Usuario.php:

namespace app\modules\vendas\models;

use Yii;
use yii\db\ActiveRecord;
use yii\web\IdentityInterface;
use yii\behaviors\TimestampBehavior;
use yii\db\Expression;

class Usuario extends ActiveRecord implements IdentityInterface
{
    // ... (outros métodos)
    
    /**
     * Busca identidade por ID
     * 🔧 AJUSTADO para validar UUID
     */
    public static function findIdentity($id)
    {
        // Validar se o ID é um UUID válido
        if (!self::isValidUuid($id)) {
            // Se não for UUID válido, fazer logout e retornar null
            if (!Yii::$app->user->isGuest) {
                Yii::$app->user->logout();
            }
            return null;
        }
        
        return static::findOne($id);
    }
    
    /**
     * Valida se uma string é um UUID válido
     * 🔧 NOVO MÉTODO
     */
    protected static function isValidUuid($uuid)
    {
        if (!is_string($uuid)) {
            return false;
        }
        
        // Padrão UUID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
        $pattern = '/^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i';
        return preg_match($pattern, $uuid) === 1;
    }
    
    // ... (resto do código)
}
✓ O que esse código faz:
• Valida se o ID é um UUID válido antes de buscar no banco
• Se encontrar um ID inválido (como "1"), faz logout automaticamente
• Evita o erro no PostgreSQL
• Usuário será redirecionado para fazer login novamente

🛡️ Prevenção (Middleware de Validação)

3Criar Middleware para Validar Sessões

Adicione um behavior no config/web.php para validar sessões automaticamente.

// Em config/web.php, adicione dentro de 'components':

'user' => [
    'identityClass' => 'app\modules\vendas\models\Usuario',
    'enableAutoLogin' => true,
    'loginUrl' => ['login/index'],
    'authTimeout' => 1800,
    // 🔧 NOVO: Validação de sessão
    'on beforeLogin' => function ($event) {
        // Se o ID não for UUID, cancela o login
        $pattern = '/^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i';
        if ($event->identity && !preg_match($pattern, $event->identity->id)) {
            $event->isValid = false;
            Yii::$app->session->destroy();
        }
    },
],

🔄 Migração de Dados (Opcional)

⚠️ Se você quer manter os usuários existentes:
Você precisa migrar os dados da tabela antiga users para prest_usuarios convertendo IDs inteiros em UUIDs.

Script de Migração SQL:

-- 1. Inserir usuários antigos na nova tabela com UUIDs novos
INSERT INTO prest_usuarios (id, nome, email, hash_senha, data_criacao, data_atualizacao)
SELECT 
    uuid_generate_v4() as id,  -- Gera UUID novo
    username as nome,           -- Ajuste conforme sua estrutura
    email,
    password_hash as hash_senha, -- Ajuste o nome da coluna
    created_at as data_criacao,  -- Ajuste conforme sua estrutura
    updated_at as data_atualizacao
FROM users
WHERE email NOT IN (SELECT email FROM prest_usuarios); -- Evita duplicatas

-- 2. Verificar quantos foram migrados
SELECT COUNT(*) FROM prest_usuarios;
⚠️ IMPORTANTE:
• Todos os usuários precisarão fazer login novamente
• Os IDs serão diferentes (novos UUIDs)
• Senhas serão mantidas se usar o mesmo algoritmo de hash
