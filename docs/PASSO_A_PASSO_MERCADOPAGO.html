Split de Vendas Mercado Pago

Guia interativo passo a passo para configurar e ativar o recebimento automático de taxas da plataforma.

1. Criar App no MP
2. Configurar SaaS
3. Conectar Lojista
4. Receber Split
Passo 1

Criar a Integração no Mercado Pago

1. Acessar o Portal de Developers

Entre no site Mercado Pago Developers utilizando a conta que deseja definir como a **dona da plataforma** (a conta que receberá os 0,5% de taxa).

2. Criar Aplicação

Vá em **Suas Aplicações** e clique em **Criar nova aplicação**. Preencha o formulário:

  • Nome da aplicação: Ex: Pulse Plus Plataforma
  • Tipo de solução de pagamento: Selecione Checkout Pro ou Pagamentos Online.
  • Modelo de integração: Selecione Marketplace / Split.

3. Configurar Credenciais de Produção e Homologação

Após criar, no menu esquerdo vá em **Credenciais** e anote:

  • Client ID (App ID): Um código numérico.
  • Client Secret (Access Token Master): Começa com APP_USR-....

4. Configurar as URLs e Callback OAuth

Na sua aplicação do Mercado Pago, configure o campo Redirect URI (URL de redirecionamento) para o endpoint de callback do sistema em produção:

https://seudominio.com.br/index.php/api/mercado-pago/oauth-callback
Passo 2

Configurar as Credenciais no Servidor (SaaS)

1. Editar o arquivo de Ambiente (.env)

Acesse o servidor de hospedagem e insira os dados gerados no Passo 1 no arquivo .env localizado na raiz do projeto:

# Configurações do Mercado Pago (OAuth e Plataforma)
MP_APP_ID="INSIRA_O_APP_ID_AQUI"
MP_CLIENT_SECRET="INSIRA_O_CLIENT_SECRET_DA_SUA_CONTA_AQUI"
MP_REDIRECT_URI="https://seu-dominio-saas.com/index.php/api/mercado-pago/oauth-callback"

2. Definir a Taxa de Comissão Padrão

No arquivo config/params.php, configure a taxa que a plataforma reterá por venda (o padrão é 0,5%):

'pulse_platform_fee_percent' => 0.005, // 0.005 representa 0.5%

Importante:

Para aplicar e atualizar as variáveis de ambiente em produção, reinicie seu servidor web ou servidor de containers caso necessário.

Passo 3

Como o Lojista Conecta sua Conta

1. Acesso do Lojista ao Painel de Integrações

O lojista cadastrado no SaaS deve acessar seu respectivo Painel Administrativo -> **Configurações de Gateway** e selecionar **Mercado Pago**.

2. Fluxo OAuth

O lojista verá o botão "Conectar com Mercado Pago". Ao clicar:

  • Ele será redirecionado para a tela oficial de login e consentimento do Mercado Pago.
  • Ele autoriza a aplicação do SaaS a gerenciar pagamentos.
  • O Mercado Pago redirecionará de volta para o sistema no endpoint oauth-callback.

3. Gravação Automática de Tokens

O sistema do SaaS captura o token do lojista (`access_token` individual) e grava na tabela do banco de dados para utilizá-lo nas futuras requisições de checkout de seus clientes de forma isolada e segura.

Passo 4

Como Funciona a Receita do Split nas Vendas

O Fluxo da Transação de Checkout

Quando um cliente compra no PWA (Catálogo) ou o vendedor realiza uma venda direta:

  1. O sistema envia a transação para o Mercado Pago vinculada ao token do lojista.
  2. O parâmetro application_fee / marketplace_fee é anexado no payload contendo o cálculo da comissão de 0.5%.
  3. O cliente realiza o pagamento.
  4. O Mercado Pago realiza a divisão (Split) imediata no ato da aprovação:
    • O valor líquido menos a taxa vai para a carteira digital do lojista.
    • Os 0,5% caem diretamente no saldo da conta do administrador da plataforma.

Conciliação Financeira Local

Toda movimentação do split também é registrada na tabela local de auditoria financeira saas_financial_logs e inserida como saída no caixa para garantir conciliação perfeita para o lojista.
